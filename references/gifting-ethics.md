# Gifting Ethics

Principles for making a gift *for someone else*, distilled from the original `relationship-gifting` module. These apply to every gift produced by this skill.

The rule underneath all of them: **the recipient is a full person, not a character the skill invents from fragments.**

## Core Principles

1. **Never feel like a CRM.** This is not contact management. Do not build structured profiles of people. Do not ask the user to fill in hobbies, aesthetic preferences, birthdays, or personality traits unless the user volunteers them. The skill works from a single concrete moment, not from a dossier.

2. **Evidence-based, always.** Every specific detail in the gift — a named place, a quoted line, a pet name, a color TA likes, an event TA attended — must trace to something the user actually shared during intake. Never invent facts about the recipient. "Probably likes..." "maybe enjoys..." are guesses dressed up as knowledge; in a gift they read as *someone pretending to know TA.*

   If the material only supports a small, specific detail, build the gift around that small thing. A gift with one true detail beats a gift with five invented ones.

3. **Soft language about the recipient.** When the gift speaks *about* TA, use possibility words: 可能 / 也许 / 说不定 / 不知道 / 好像. When it speaks *to* TA, ground in shared moments: "那天你说的..." not "你是一个...的人." The skill is offering a thought, not issuing a diagnosis.

4. **Respect silence.** The skill produces the gift; the user sends it. If the recipient never responds, that IS the response. Do not write the gift in a way that demands a reply ("等你回复" / "告诉我你的答案"). Let the gift be complete as a one-way gesture.

5. **The skill never contacts the recipient directly.** It only produces an artifact. The user delivers it, by whatever channel fits the relationship.

## Evidence Rule in Practice

Before finalizing any specific line or visual detail, ask: *where in the intake material does this come from?*

| Type of detail | Acceptable source | Unacceptable source |
|---|---|---|
| A specific place ("那家巷子里的面馆") | User mentioned it; shows in a photo | Invented to add flavor |
| A quoted line from TA | Visible in a chat excerpt or attributed by user | Composed by the skill in TA's voice |
| A trait ("TA 是一个很温柔的人") | Repeatedly evident across multiple pieces of material | Inferred from a single photo or a stereotype about the relationship type |
| A shared memory | User described it | Reconstructed from a hint |
| An aesthetic preference | User said it or visibly consistent across TA's own posts | Guessed from relationship type ("妈妈应该喜欢暖色") |

If a gift detail cannot be traced back to the material, either cut it or replace it with something that can be traced. The creative power of the skill comes from *finding the right anchor*, not from *inventing plausible flavor*.

## Privacy Boundaries

Not everything the user shares about the recipient belongs in the gift.

**Safe to use in the gift:**

- Lighthearted or fun content the user shared about TA
- Creative work, hobbies, or interests the recipient is clearly proud of
- Positive shared memories
- Content TA posted publicly (朋友圈, small red book, Instagram) — treat as TA's own expression
- Specific factual details that honor TA

**Do NOT use in the gift:**

- The user's private venting about their own struggles (that is the user's context, not the recipient's)
- The user's complaints *about the recipient themselves* ("我妈又唠叨了") — use this to understand the relationship, never to land a point at TA's expense
- Relationship problems between the user and the recipient — if the gift is clearly trying to repair something strained, the skill should make the *repair gesture* itself the anchor, not reference the conflict
- Information TA shared with the user privately that TA might not want visible in an artifact TA could re-share
- Anything that could embarrass TA if TA showed the gift to a third party

When in doubt: **don't include it**. The gift should be something TA would be happy to receive, show friends, or keep. If there's any chance a detail would make TA uncomfortable, cut it.

## Soft Language Examples

Good (soft, evidence-based, casual):

> "你说你最近开始学陶艺——不知道你桌上现在有没有多一个歪歪扭扭的杯子？"

> "那张照片里 TA 笑得挺松弛的，不知道那天是什么让 TA 开心。"

> "可能 TA 已经忘了这件事了，但那次你提到 TA 说 ___，挺打动人的。"

Bad (too formal, guilt-trippy, baseless):

> ❌ "根据您提供的信息，推测她是一位热爱生活的女性。"

> ❌ "你好久没给妈妈送东西了，要不要考虑一下？"  

> ❌ "妈妈一定会喜欢这个的。"（断言 / 凭空）

> ❌ "她可能正在等你的消息。"（guilt）

## When the Gift Speaks Directly to the Recipient

Some gifts are written in the second person, addressed to TA (a letter, a note, an image with TA's name on it). For these:

- Use the language and register the user typically uses with TA (not the skill's default voice)
- Never claim internal states the skill can't verify ("我知道你最近很累" — do you?)
- If a line would feel presumptuous if the user's mother / friend / partner saw it out of context, cut it
- Include a specific, traceable detail — that is how the gift earns the right to speak directly

## When the Gift is About TA but Not Addressed to TA

Some gifts are made *for the user to give*, but the content is in the third person or abstracted (an image of TA's hobby, a collage of places TA has been). For these:

- Still apply the evidence rule
- Lean even more on visual / aesthetic resonance rather than spoken claims
- The user should be able to hand this over and say "I made this for you" without the gift itself putting any words in TA's mouth

## Cross-Cultural Sensitivity

When the recipient belongs to a different generation, culture, or social context than the user:

- Default to more restraint, not more creativity. A modest, warm gift will translate; an avant-garde concept may land as alienating.
- Avoid inside jokes or memes unless the intake material clearly shows TA engages with that style.
- Formal relationships (a senior colleague, a mentor) usually want *text* or *image*, not *text-play*. Text-play assumes synchronous playfulness the recipient may not have time or inclination for.

## The "Would TA Want This?" Test

Before delivering, hold the final gift up against one question:

> Would the *recipient*, seeing this, feel warmed / seen / delighted — or would they feel slightly uncomfortable that the user seems to know this much about them?

If the answer tilts toward the second, the gift has likely crossed from *thoughtful* into *surveillant*. Pull back on specificity, soften language, remove any detail that feels like it was extracted rather than shared.

## What to Do With Sparse Material

If the user's material is thin (a single photo, a one-line quote, a relationship type with no specifics):

- Do NOT try to compensate by inventing richness
- Do make a *smaller, more honest* gift — `light` weight, anchored on the one true thing the material gave you
- A sincere `light` gift built on one real detail lands harder than a `standard` gift built on five guesses

This is the most common failure mode of gift skills: fake depth in the absence of real context. Better to be small and true.
